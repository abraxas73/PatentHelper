### 변경 사항

#### 1. 도면 인식 로직 단순화
**파일**: `app/core/pdf_processor.py`

##### `_is_drawing_page()` 메서드
- **변경 전**:
  - 복잡한 조건들을 종합적으로 판단
  - 이미지 존재 여부, 텍스트 밀도, 그래픽 요소, 도면 키워드 등 다양한 기준 사용
  - 약 80줄의 복잡한 로직

- **변경 후**:
  - 단순히 "도면[숫자]" 패턴만 확인
  - 정규표현식: `r'도면\s*\d+'`
  - 약 10줄로 간소화

##### `_find_drawing_area_precise()` 메서드
- **변경 전**:
  - 페이지 내 그래픽 요소를 분석하여 도면 영역 계산
  - 상하단 여백 자동 추가 (페이지 높이의 10% + 80px)
  - 텍스트 헤더 영역 제외 로직 포함
  - 약 150줄의 복잡한 분석 로직

- **변경 후**:
  - 전체 페이지를 도면 영역으로 반환
  - 여백 고려 없음
  - 3줄로 간소화

### 변경 이유
- 사용자 요구사항: "도면1", "도면2" 등의 텍스트가 명시적으로 포함된 페이지만 도면으로 인식
- 페이지별 상하단 여백 고려 불필요
- 더 정확하고 예측 가능한 도면 추출
