# 변경 이력

## [2025.09.19] - 양방향 회전 감지 및 PDF 병합 개선

### ✨ 새로운 기능
- **양방향 회전 감지 (+90°/-90°)**
  - OCR 인식률 향상을 위해 양방향 회전 시도
  - +90도 시계방향과 -90도 반시계방향 모두 검사
  - 가장 많은 번호를 감지한 방향 선택
  - 회전 상태별 독립적 OCR 수행

- **개선된 도면 페이지 감지**
  - 독립 라인 패턴 매칭 방식 도입
  - 도면 번호가 단독 라인에 있는 경우만 도면으로 인식
  - 텍스트 페이지에서 도면 참조만 있는 경우 제외
  - 실제 도면 페이지만 정확하게 추출

### 🐛 버그 수정
- **회전 복원 문제 해결**
  - PIL.Image.rotate() 방향 이해 오류 수정
  - PIL은 양수가 반시계방향임을 반영
  - +90° 회전 후 복원 시 PIL에서 90도로 복원
  - -90° 회전 후 복원 시 PIL에서 -90도로 복원
  - 180도 뒤집힘 문제 완전 해결

- **PDF 병합 실패 처리**
  - 상세한 에러 로깅 추가
  - 병합 실패 시 원인 파악 가능
  - 도면만 포함 PDF와 완성 PDF 구분 명확화

### 🔧 기술적 개선
- scipy 의존성 제거 (numpy만 사용하여 엔트로피 계산)
- batch_annotate 메서드 시그니처 수정 (rotation_status_by_image 파라미터 추가)
- UI 텍스트 개선: "어노테이션 PDF" → "완성 PDF 다운로드"

## [2025.09.19] - 도면 콘텐츠 검증 및 회전 감지 기능 추가

### ✨ 새로운 기능
- **도면 이미지 콘텐츠 검증**
  - 추출된 도면에 실제 이미지 콘텐츠가 있는지 검증
  - 엔트로피 계산을 통한 의미있는 콘텐츠 감지
  - 최소 5% 이상 non-white 픽셀 및 엔트로피 2.0 이상 필요
  - 빈 페이지나 텍스트만 있는 페이지 필터링

- **90도 회전 감지 및 처리**
  - OCR 인식률 향상을 위한 자동 회전 감지
  - 원본 방향에서 3개 미만 번호 감지 시 90도 회전 시도
  - 회전된 상태에서 OCR 수행 후 좌표 재계산
  - 어노테이션 적용 후 원본 방향으로 자동 복원

### 🔧 기술적 개선
- pdf_processor.py: 이미지 엔트로피 및 콘텐츠 비율 계산
- image_extractor.py: find_numbered_regions_with_rotation() 메서드 추가
- image_annotator.py: is_rotated 파라미터로 회전 상태 처리
- OCR 프로세서: 회전 메타데이터 전달 및 처리
